## Как просмотреть ошибки в консоли разработчика

* Нажмите сочетание клавиш в окне Cherry Studio: <kbd>Ctrl</kbd> + <kbd>Shift</kbd> + <kbd>I</kbd> (Mac: <kbd>Command</kbd> + <kbd>Option</kbd> + <kbd>I</kbd>)

{% hint style="info" %}
- Окно Cherry Studio должно быть активным для вызова консоли;
- Консоль нужно открыть до отправки запроса (диалог, тест и т.д.).
{% endhint %}

* В консоли перейдите в <mark style="color:blue;">`Network`</mark> → найдите последний запрос с красным <mark style="color:red;">`×`</mark> (<mark style="color:red;">`completions`</mark> для диалогов или <mark style="color:red;">`generations`</mark> для генерации изображений) → вкладка <mark style="color:blue;">`Response`</mark> покажет детали ошибки (область ④).

> Если не удаётся определить причину, сделайте скриншот и отправьте в [официальный чат](https://t.me/CherryStudioAI).

{% endhint %}

## Формула не отображается / Ошибка рендеринга

* Если отображается код формулы вместо результата — проверьте разделители:

> **Синтаксис разделителей**
>
> _Встроенные формулы_
>
> * Одинарные доллары: `$формула$`
> * Или: `\(формула\)`
>
> _Отдельные блоки_
>
> * Двойные доллары: `$$формула$$`
> * Или: `\[формула\]`
> * Пример: `$$\sum_{i=1}^n x_i$$`\
>   $$\sum_{i=1}^n x_i$$

* Искажённый рендеринг/кракозябры (особенно с китайскими символами) — переключите движок рендеринга на KateX.

## Модель не распознаёт изображения / Ошибка загрузки

Сначала проверьте поддержку vision-функционала — в Cherry Studio соответствующие модели помечены иконкой 👁️.

Vision-модели поддерживают загрузку изображений. Если функциональность не определена автоматически:
- Найдите модель в списке провайдера
- Нажмите иконку настроек рядом с названием
- Активируйте опцию "Изображения"

Неподдерживающие изображения модели не смогут обрабатывать картинки даже при включённой опции. Проверяйте документацию провайдера по конкретным моделям.
