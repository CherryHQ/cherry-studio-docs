# Настройки провайдеров

На этой странице представлено только описание функционала интерфейса. Руководство по настройке можно найти в базовом руководстве в разделе [Настройка провайдеров](../../../pre-basic/providers/).

{% hint style="info" %}
* При использовании встроенных провайдеров достаточно указать соответствующий API-ключ.
* Разные провайдеры могут использовать различные термины для ключа: секретный ключ, Key, API Key, токен - все это означает одно и то же.
{% endhint %}



### API-ключи

В Cherry Studio один провайдер поддерживает ротацию нескольких ключей. Ротация осуществляется циклически по порядку ключей в списке.

* Несколько ключей добавляются через запятую на английском языке. Пример:

<pre><code><strong>sk-xxxx1,sk-xxxx2,sk-xxxx3,sk-xxxx4</strong></code></pre>

{% hint style="warning" %}
Обязательно используйте запятую на **английском** языке.
{% endhint %}

### API-адрес

При работе со встроенными провайдерами обычно не требуется указывать API-адрес. При необходимости изменения строго следуйте адресу, указанному в официальной документации провайдера.

> Если провайдер предоставляет адрес вида <mark style="background-color:red;">https://xxx.xxx.com</mark><mark style="background-color:green;">/v1/chat/completions</mark>, укажите только корневую часть (<mark style="background-color:red;">https://xxx.xxx.com</mark>).
>
> Cherry Studio автоматически добавит оставшийся путь (<mark style="background-color:green;">/v1/chat/completions</mark>). Несоблюдение этого правила может привести к ошибкам работы.

{% hint style="info" %}
Уточнение: Большинство провайдеров используют стандартные пути для языковых моделей, поэтому обычно не требуется дополнительных действий. Если путь API отличается (v2, v3/chat/completions и т.д.), введите соответствующую версию с символом `/` в конце. Для нестандартных путей (отличных от <mark style="background-color:green;">/v1/chat/completions</mark>) используйте полный адрес с символом `#` в конце.

То есть:
* Адреса с `/` в конце: добавление "<mark style="background-color:green;">chat/completions</mark>"
* Адреса с `#` в конце: использование полного адреса без изменений.

{% endhint %}



### Добавление моделей

Обычно при нажатии кнопки `Управление` в левом нижнем углу страницы настроек провайдера автоматически отображаются все доступные модели. Нажмите `+` напротив нужной модели для добавления в список.

{% hint style="info" %}
Модели в появившемся списке не добавляются автоматически. Для отображения в списке выбора моделей необходимо нажать `+` и перенести модель в основной список конфигурации.
{% endhint %}


### Проверка подключения

Нажмите значок проверки возле поля ввода API-ключа для тестирования конфигурации.

{% hint style="info" %}
Проверка использует последнюю диалоговую модель из списка. При неудачной проверке убедитесь, что в списке нет неподдерживаемых моделей.
{% endhint %}

{% hint style="danger" %}
После успешной настройки ОБЯЗАТЕЛЬНО активируйте переключатель в правом верхнем углу. Без этого провайдер останется неактивным, и его модели не появятся в списке выбора.
{% endhint %}
