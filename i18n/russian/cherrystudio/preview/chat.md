## Ассистенты и темы

### Ассистент

`Ассистент` — это персонализированные настройки выбранной модели для её использования, такие как предустановленные подсказки и параметры. Эти настройки помогают адаптировать модель под ваши рабочие ожидания.

`Ассистент по умолчанию` содержит универсальные параметры (без подсказок). Вы можете использовать его напрямую или найти нужные предустановки на странице [Агентов](agents.md).

### Темы

`Ассистент` является родительским элементом для `тем`. Под одним ассистентом можно создать несколько тем (диалогов), где все `темы` используют общие настройки модели: параметры, предустановленные подсказки (prompt) и т.д.

### Настройки модели

Параметры синхронизируются с настройками ассистента. См. раздел [Настройки ассистента](chat.md#редактирование-ассистента).

{% hint style="info" %}
В настройках диалога только эти параметры применяются к текущему ассистенту. Остальные — глобальные. Например: стиль сообщений (баблы) действует во всех темах.
{% endhint %}

## Настройки ассистента

В интерфейсе ассистента → клик правой кнопкой на имени → выберите настройку

### Редактирование ассистента

{% hint style="info" %}
Настройки применяются ко всем темам ассистента.
{% endhint %}

<figure><img src="../../.gitbook/assets/image (6) (1) (1).png" alt=""><figcaption></figcaption></figure>

#### Настройки подсказок

#### <mark style="color:blue;">**`Название`**</mark>：

Настраиваемое имя для удобства идентификации.

#### <mark style="color:blue;">**`Подсказка`**</mark>：

Поле prompt. Можно редактировать по примеру из раздела агентов.

#### Настройки модели

#### <mark style="color:blue;">**`Модель по умолчанию`**</mark>：

Фиксирует модель по умолчанию для ассистента. Без настройки используется глобальная модель (см. [Модель ассистента по умолчанию](settings/default-models.md#модель-ассистента-по-умолчанию)).

{% hint style="info" %}
Локальная модель ассистента имеет приоритет над глобальной.
{% endhint %}

#### <mark style="color:blue;">**`Автосброс модели`**</mark>：

Включено: при создании новой темы модель сбрасывается к настройкам ассистента.
Выключено: новая тема наследует модель предыдущей темы.

> Пример:
> Модель по умолчанию: gpt-3.5-turbo. В теме 1 переключились на gpt-4o.
> * Автосброс включен → тема 2: gpt-3.5-turbo
> * Автосброс выключен → тема 2: gpt-4o

#### <mark style="color:blue;">**`Температура (Temperature)`**</mark> ：

Управляет случайностью вывода (по умолчанию 0.7):

* Низкие значения (0-0.3): точные ответы (код, анализ)
* Средние (0.4-0.7): баланс для диалогов
* Высокие (0.8-1.0): креативность (но может снижать связность)

#### <mark style="color:blue;">**`Top P (ядерная выборка)`**</mark>：

По умолчанию 1. Меньшие значения → консервативные ответы, большие → разнообразие:

* Малые (0.1-0.3): технический контент
* Средние (0.4-0.6): стандартные задачи
* Большие (0.7-1.0): креативные задачи

{% hint style="info" %}
Параметры можно комбинировать. Рекомендуется экспериментальная настройка под конкретные задачи.
{% endhint %}

#### <mark style="color:blue;">**`Размер контекста (Context Window)`**</mark>

Количество сохраняемых сообщений в контексте:

* 5-10: стандартные диалоги
* >10: сложные задачи (генерация длинных текстов)
* Больше сообщений → больше токенов

#### <mark style="color:blue;">**`Ограничение длины ответа (MaxToken)`**</mark>

Максимальное количество токенов в ответе. Влияет на полноту вывода.

> Пример: для проверки подключения модели можно установить MaxToken=1

Большинство моделей поддерживают до 32k токенов (уточняйте в документации).

{% hint style="success" %}
Рекомендации:
* Диалоги: 500-800
* Короткие тексты: 800-2000
* Генерация кода: 2000-3600
* Длинные тексты: 4000+ (требует поддержки модели)
{% endhint %}

{% hint style="warning" %}
При слишком низком MaxToken ответы могут обрезаться.
{% endhint %}

#### <mark style="color:blue;">**`Потоковый вывод (Stream)`**</mark>

Включено: вывод по мере генерации ("эффект пишущей машинки").
Выключено: вывод полного ответа единым блоком.

{% hint style="info" %}
Отключайте для моделей без поддержки потокового вывода (например, ранние версии o1-mini).
{% endhint %}

#### <mark style="color:blue;">**`Пользовательские параметры`**</mark>

Дополнительные параметры запроса (например, `presence_penalty`). Требует знания API модели.

> Пример: top-p, maxtokens уже встроены.

Синтаксис: имя_параметра—тип—значение. См. документацию: [ссылка](https://openai.apifox.cn/doc-3222739)

{% hint style="info" %}
* Пользовательские параметры имеют приоритет над встроенными.
* Используйте <kbd>имя_параметра:undefined</kbd> для исключения параметра.
{% endhint %}
