# 音声機能

```
Cherry Studio 音声機能の利用説明書

一、音声機能の概要

Cherry Studio は三つの主要な音声機能モジュールを提供します：TTS（テキスト読み上げ）、ASR（音声認識）、音声通話。これらの機能により、音声でAIと自然に対話し、使用体験を向上させることができます。

- TTS（テキスト読み上げ）：AIの応答テキストを音声に変換
- ASR（音声認識）：ユーザーの音声をテキスト入力に変換
- 音声通話：TTSとASRを組み合わせ、ChatGPTのような音声対話体験を実現

二、TTS（テキスト読み上げ）機能

1. 対応サービス種類

Cherry Studio は四種類のTTSサービスに対応しています：

- OpenAI：OpenAIのTTS APIを利用（APIキーが必要）
- ブラウザTTS：ブラウザ組み込みの音声合成機能（無料・設定不要）
- 硅基流動（Siliconflow）：SiliconflowのTTSサービス（APIキーが必要）
- 無料オンラインTTS：無料のオンラインTTSサービス（APIキー不要）

2. 設定方法

1) 設定ページで「音声機能」タブを選択
2) 「TTS」サブタブで：
   - TTS機能を有効化（トグルをオン）
   - TTSサービス種類を選択
   - 選択したサービスに応じたパラメータを設定：
     - OpenAI：APIキー、APIアドレス、音声タイプとモデルを設定
     - ブラウザTTS：音声タイプを選択
     - 硅基流動：APIキー、APIアドレス、音声タイプ、モデル、応答フォーマット、話速を設定
     - 無料オンラインTTS：音声タイプと出力フォーマットを選択
3) TTSフィルター設定（任意）：
   - 思考過程を除外
   - Markdown記号を除外
   - コードブロックを除外
4) TTS進捗バーの表示設定
5) 「TTSテスト」ボタンで設定を確認

3. 使用方法

- TTS機能有効時、AI応答が自動的に音声出力される
- チャット画面で各AI応答下部にTTS再生ボタンが表示
- 再生ボタンクリックで音声を再生/一時停止
- TTS進捗バーが有効ならテキスト下部に進捗が表示
- 長文は自動分割され連続再生される

三、ASR（音声認識）機能

1. 対応サービス種類

Cherry Studio は三種類のASRサービスに対応：

- OpenAI：OpenAIのWhisperモデル（APIキー必要）
- ブラウザ：ブラウザ組み込み音声認識（無料・設定不要）
- ローカルサーバー：WebSocket経由のローカル音声認識

2. 設定方法

1) 設定ページで「音声機能」タブを選択
2) 「ASR」サブタブで：
   - ASR機能を有効化（トグルをオン）
   - ASRサービス種類を選択
   - 選択したサービスに応じた設定：
     - OpenAI：APIキー、APIアドレス、モデルを設定
     - ブラウザ：追加設定不要
     - ローカルサーバー：アプリ起動時のサーバー自動起動を設定可能
   - 音声認識言語を選択（デフォルトは中国語）
3) 「ASRテスト」ボタンで設定確認

3. 使用方法

- ASR有効時、入力欄横に音声認識ボタンが表示
- ボタンクリックで録音開始
- 発話後、音声がテキスト変換され入力欄に反映
- 再度クリックで録音終了
- 連続複数発話に累積モードで対応

四、音声通話機能

1. 機能特徴

- TTSとASRを統合したChatGPT型音声対話
- ドラッグ可能なフローティングウィンドウ
- 長押し発話モード対応
- カスタムホットキー設定可能
- ウィンドウ折りたたみ機能
- 専用通話モデル選択可能
- カスタムプロンプト対応

2. 設定方法

1) 設定ページで「音声機能」タブを選択
2) 「通話機能」サブタブで：
   - 音声通話機能を有効化（トグルをオン）
   - 「モデル選択」ボタンで通話用AIモデルを選択
   - プロンプト欄でカスタムプロンプトを設定（任意）
   - 「保存」でプロンプト適用、「リセット」でデフォルトに復元

3. 使用方法

1) チャット画面で入力欄右側の通話ボタン（電話アイコン）をクリック
2) 通話ウィンドウが開き、歓迎音声が再生
3) 「長押しで発話」ボタンを長押しで録音開始（ホットキーも可）
4) ボタン離すと送信されAI処理
5) AI応答がTTSで再生
6) ウィンドウ操作ボタン：
   - ミュート解除：TTS出力制御
   - 一時停止再開：対話制御
   - 設定：ホットキー設定
   - 折りたたみ：簡易表示切り替え
7) 閉じるボタンで通話終了

4. ホットキー設定

1) 通話ウィンドウの設定ボタンクリック
2) 設定パネルで「ホットキー」ボタン選択
3) 設定したいキーを押下（スペースキー、Shiftキー等）
4) 「保存」で設定適用
5) 使用時はホットキー長押しで録音、離すと送信

五、よくある問題と解決策

1. TTS関連問題

- 問題：音声出力なし
  解決策：TTS有効状態、サービス設定、パラメータを確認

- 問題：音声品質不良
  解決策：サービス種類や音声タイプの変更を試す

- 問題：エラー表示
  解決策：APIキー、ネットワーク接続を確認

2. ASR関連問題

- 問題：音声認識不可
  解決策：ASR有効状態、サービス設定を確認

- 問題：認識精度低下
  解決策：サービスの切り替え、マイク位置や音量調整

- 問題：サーバー接続失敗
  解決策：ローカルサーバーの稼働状態確認、アプリ再起動

3. 通話関連問題

- 問題：通話ウィンドウ起動不可
  解決策：通話機能有効状態、TTS/ASR設定を確認

- 問題：長押し反応なし
  解決策：マイク権限設定を確認、通話再起動

- 問題：AI応答が音声出力されない
  解決策：TTS有効状態、ミュート状態を確認

六、高度な設定とカスタマイズ

1. TTS詳細設定

- フィルター：思考過程/Markdown/コードブロック除外でスムーズな再生
- 進捗バー表示切替
- カスタム音声タイプとモデル追加

2. ASR詳細設定

- サーバー自動起動設定
- 言語選択（複数対応）

3. 通話詳細設定

- カスタムプロンプト：音声通話時のAI応答を最適化
- 専用モデル選択：通常チャットと異なるモデルを割り当て
- カスタムホットキー：録音操作を個人設定

七、利用のアドバイス

1. TTSサービス選択：
   - 高品質音声なら「OpenAI」または「硅基流動」
   - API設定不要なら「ブラウザTTS」または「無料オンラインTTS」

2. ASRサービス選択：
   - 高精度認識なら「OpenAI」
   - API設定不要なら「ブラウザ」機能

3. 音声通話の最適化：
   - ヘッドホン使用でTTS出力の再録音を防止
   - 静かな環境で認識精度向上
   - カスタムプロンプトで音声出力に最適化

4. ニーズに応じた設定：
   - テキスト主体ならTTSのみ有効化
   - 音声入力主体ならASRのみ有効化
   - 完全音声対話なら通話機能を有効化
```
