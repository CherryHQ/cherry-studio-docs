# 5 分钟搞懂：助手 / 智能体 / 技能 / MCP / 频道

第一次打开 Cherry Studio，看到「助手」「智能体」「技能」「MCP」「频道」「记忆」这些名字，是不是有点懵 —— 都是 AI，都听起来差不多，到底是啥关系？

这一页用最朴素的话告诉你**它们各自是什么、解决什么问题、什么时候用哪个**，看完你就能心里有数。

## 一图先抓住全貌

把 Cherry Studio 想象成一家公司：

* **模型服务（Provider）** = 给员工大脑的"能源公司"（OpenAI、Claude、DeepSeek 这些卖电的）
* **助手（Assistant）** = 一个带固定人设的 **AI 同事**，比如"严谨的代码 reviewer"、"温柔的写作老师"
* **智能体（Cherry Agent）** = 升级版同事，**有手脚** —— 会自己读文件、查资料、跑命令，能完成多步骤任务
* **技能（Skill）** = 装在同事身上的"专业能力包"，比如"会做 PPT"、"会写专利申请"，开箱即用
* **MCP** = 同事用来**插外部工具**的"USB 接口"，可以接你的本地文件、数据库、Notion 笔记等等
* **频道（Cherry Claw）** = 把同事派出去到**飞书/微信/Telegram** 当群机器人上班
* **定时任务** = 给同事一份**值班表**，让它按点自动干活
* **全局记忆** = 同事的**长期记忆本**，记住你的偏好和上下文

## 对比表：什么时候用哪个？

| 你想做的事 | 推荐用 |
|---|---|
| 跟 AI 聊聊天，让它**用某种风格回复**（比如"温柔耐心"、"专业严谨"） | **助手** |
| 让 AI **自己读文件**、**自己执行命令**、**完成多步任务**（比如"帮我整理这个项目代码并写文档"） | **智能体（Cherry Agent）** |
| 想给 AI 装个**专门技能**（"会做小红书图文"、"会画流程图"） | **技能** |
| 让 AI 能**用我电脑上的工具**（读本地文件、查数据库、抓网页等） | **MCP** |
| 让我的 AI **在飞书群里给同事干活** | **频道（Cherry Claw）** |
| 让 AI **每天早上 9 点自动**给我做新闻简报 | **定时任务**（搭配 Agent） |
| 让 AI **记住我是谁、我喜欢什么** | **全局记忆** |

## 逐个说人话

### 1. 助手（Assistant）—— 给 AI 套一身"人设衣服"

普通对话像"白板"，每次开新窗口都得从头跟 AI 说"你是 xxx，请按 xxx 风格回复"。

**助手** = 把这身"人设衣服"**永久存下来**，下次打开直接用，省去重复设定。一个助手 = 一份"系统提示词 + 模型参数预设"。

* **怎么用**：顶部 Tab `+` → 启动台 → `助手库` 浏览、添加或自建
* **可以叠加**：助手可以挂载知识库、技能、工具
* **细节文档**：[智能体（助手广场）](../cherrystudio/preview/agents.md)

### 2. 智能体 / Cherry Agent —— 给 AI 装"手脚"

普通对话里 AI 只会"嘴上说"，要它真去读文件、跑命令、做事，它做不到。

**Cherry Agent** = 一个**会自己动手**的升级版 AI。你给它一个目标（"把这个目录下所有 PDF 整理成 Excel"），它会自己拆解步骤、调用工具、一步步执行、遇到问题自己想办法解决。

* **怎么用**：顶部 Tab `智能体` → 创建一个 Agent
* **前置**：需要先开 [API 服务器](api-server.md) + 配一个支持 Anthropic 协议的 [Provider](../pre-basic/providers/)（如 [CherryIN](../pre-basic/providers/cherryin-1.md) / [Anthropic](../pre-basic/providers/anthropic.md)）
* **细节文档**：[Cherry Agent](agent.md)

{% hint style="info" %}
**助手 vs Cherry Agent 一句话区分**：
* 助手 = 只会**说**的同事（聊天）
* 智能体 = 会**动手**的同事（执行任务）
{% endhint %}

### 3. 技能（Skill）—— 给 AI 装"专业能力 App"

智能体很聪明，但默认是"通才"。如果你经常做某类专门任务（写小红书图文、做 PPT、做财务分析），可以装个**对应技能**。

类比：**给你手机装 App**。微信能聊天、抖音能刷视频，技能就是给 AI 装"专门软件"。

* **怎么用**：`设置 → 技能` 浏览/安装，然后在助手或 Agent 设置中勾选启用
* **来源**：内置技能 + 社区/官方技能市场
* **细节文档**：[技能](../pre-basic/settings/skills.md)

### 4. MCP —— AI 用的"USB 接口"

智能体要让 AI 自己动手做事，但 AI 自己什么都没有 —— 它**怎么读你电脑上的文件**？**怎么查你公司数据库**？

**MCP（Model Context Protocol）** = AI 用的**统一接口标准**。任何开发者按这个标准做一个"小工具"（叫 MCP Server），你的 AI 就能用上它。

类比：手机的 **USB / Type-C 接口** —— 充电器、U 盘、键盘、摄像头都能插，因为大家都遵守同一个接口规范。AI 也一样，有了 MCP，它就能"插入"各种外部工具。

* **怎么用**：`设置 → MCP 服务器` 添加你想接入的 MCP Server（如"读本地文件"、"查 Notion 笔记"、"操作 GitHub"等）
* **谁能用**：智能体和（部分）助手可以挂载 MCP
* **细节文档**：[MCP 使用教程](mcp/)

{% hint style="info" %}
**技能 vs MCP 一句话区分**：
* **技能** = 自带的"AI 才艺包"（怎么写、怎么画、怎么排版）—— 偏**软**
* **MCP** = 让 AI 连到"外部工具与数据"（读文件、查 DB、调 API）—— 偏**硬**

类比：技能像你的**学历背景**（怎么思考），MCP 像你的**工具箱**（能用什么硬件）。
{% endhint %}

### 5. 频道 / Cherry Claw —— 把 AI 派去 IM 群

你训练好了一个智能体，但每次跟它说话都要打开 Cherry Studio。如果你想**让它住在飞书群里**，群成员 at 它就能让它干活呢？

**频道（Cherry Claw）** = 把你的 Cherry Agent **接到 IM 平台**当机器人。当前支持飞书 / Telegram / QQ / 微信 / Discord / Slack。

* **怎么用**：先创建好 Agent → `设置 → 频道` → 选平台 → 填该平台的机器人凭据
* **细节文档**：[频道（Cherry Claw）](agent-channels.md)

### 6. 定时任务 —— AI 的"值班表"

如果你想要"每天早上 9 点自动给我做一份昨天的新闻简报"、"每周一上午汇总团队工作量"，不用每次手动启动 —— **定时任务**让 Agent 按时间表自动跑。

* **怎么用**：`设置 → 定时任务` → 选 Agent + 设触发时间 + 写任务提示词
* **细节文档**：[定时任务](scheduled-tasks.md)

### 7. 全局记忆 —— AI 的"长期记忆本"

普通对话 AI 是"金鱼脑"，每次开新话题就忘了你是谁。

**全局记忆** = 让 AI **跨对话记住关键信息**（你的职业、偏好、长期事实），新对话也能用上。

* **怎么用**：`设置 → 全局记忆` → 开启 → 加几条关于你自己的记忆
* **细节文档**：[全局记忆](memory.md)

## 关系图（一句话版）

> **Provider** 提供大脑 → **助手 / 智能体** 是有人设的同事 → 通过**技能**学专业本事 → 通过 **MCP** 连接外部工具 → 通过**频道**派去 IM 干活 → 通过**定时任务**自动值班 → 通过**全局记忆**记住你

## 我应该按什么顺序学？

如果你是第一次用，推荐这个顺序：

1. **第一步**：先在 `设置 → 模型服务` 配好一个 [Provider](../pre-basic/providers/)，能聊天就行
2. **第二步**：去 `助手库` 添加一个你常用场景的助手（比如"翻译助手"），开始日常聊天
3. **第三步**：用上 [知识库](../knowledge-base/knowledge-base.md) 让 AI 能查你的资料
4. **第四步**：想要更自动 → 开 [API 服务器](api-server.md) → 配 [Cherry Agent](agent.md)
5. **第五步**：进阶 → [MCP](mcp/) 接工具 / [频道](agent-channels.md) 派去 IM / [定时任务](scheduled-tasks.md) 自动跑

不必一上来就全套，按需求逐步解锁即可。
