# MCP 使用教程

## 先用一句话搞懂 MCP

**MCP = 让 AI 能用你电脑/账号里东西的"统一接口"。**

打个比方：

* 你的 AI 默认像是**没装任何 App 的新手机** —— 只会聊天，看不到你电脑里的文件，登不上你的 Notion，打不开你的浏览器
* **MCP** 就像手机的**应用商店 + USB 接口** —— 装上一个 MCP，AI 就多一项本事："会读文件了"、"会查 Notion 了"、"会自动发邮件了"

这套接口由 Anthropic（做 Claude 的那家）发起做了一个统一规范，全世界开发者都按这个规范做"小工具"（叫 **MCP Server**），所以任何支持 MCP 的 AI 客户端（包括 Cherry Studio）都能用上同一批工具。

## 我用 MCP 能干什么？

举几个普通用户也用得上的例子：

* **读本机文件**：让 AI 看你 `~/Documents` 里的笔记、对账单、合同
* **接 Notion / Obsidian / Apple Notes**：让 AI 查询和修改你的笔记本
* **自动发邮件、发飞书消息**：让 AI 帮你起草 + 直接发出
* **查数据库**：让 AI 自己写 SQL 查 MySQL / PostgreSQL
* **浏览网页 + 截屏**：让 AI 操作浏览器查信息
* **代码相关**：让 AI 直接读/改本地代码仓库、查 GitHub Issue

## MCP 和「技能」有什么不一样？

很多人在这里卡住。简单说：

| | **技能（Skill）** | **MCP** |
|---|---|---|
| **是什么** | "怎么做" 的能力包 | "用什么" 的接口 |
| **形象比喻** | AI 的学历背景（思考方式） | AI 的工具箱（动手能力） |
| **典型例子** | "会做小红书图文"、"会画流程图" | "能读文件"、"能查 Notion"、"能发邮件" |
| **是否需要外部账号/工具** | 不需要（纯模型层） | 经常需要（接外部 App / API） |

**实操区别：** 装一个"小红书技能"，AI 输出会符合小红书风格；装一个 Notion MCP，AI 才**真的能**进你的 Notion。

## 常见疑问

**Q：装 MCP 是不是要写代码？**
不需要。基本上都是"复制一段配置粘进 Cherry Studio"或者"让 AI 自己装"。

**Q：装错了会把我电脑搞坏吗？**
低风险。MCP 默认只能做它声明的事（声明要读文件的就只读文件，没声明要发邮件的就发不了），但**装来历不明的 MCP 仍有风险**（毕竟是别人写的代码）。建议从官方仓库找。

**Q：MCP 会偷我数据吗？**
取决于具体的 MCP。官方仓库里的开源 MCP 可审查代码，相对放心；第三方闭源 MCP 装前请确认开发者。
